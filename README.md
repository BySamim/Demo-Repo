# SharmaAI

SharmaAI is a static Vite React app for an AI-assisted CBSE Class X maths learning platform. It presents a demo learning experience for R.D. Sharma chapters with dashboards, chapter pages, AI lesson views, answer submission, progress tracking, and an admin console.

The current app is UI-first and uses local mock data from `src/lib/mock-data.ts`.

For project guidance, validation rules, and the current done/to-do checklist, see
`STEERING.md`.

## Tech Stack

- React 19
- TanStack Router
- TanStack Query
- Vite
- Tailwind CSS v4
- shadcn-style UI primitives
- lucide-react icons

## Getting Started

Install dependencies:

```bash
npm install
```

Run the development server:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

The production build is written to `dist/` and contains static HTML, CSS, and JavaScript assets.

Preview a production build:

```bash
npm run preview
```

Lint the project:

```bash
npm run lint
```

Format files:

```bash
npm run format
```

## Static Hosting

This app is configured for static hosting from `dist/`.

For GitHub Pages or any host without single-page app rewrites, the router uses hash URLs such as:

- `/#/dashboard`
- `/#/chapters`
- `/#/lesson/real-numbers-euclid`

To deploy:

1. Run `npm run build`.
2. Publish the generated `dist/` directory.
3. Use `dist/index.html` as the static entry point.

`vite.config.ts` sets `base: "./"` so built assets work from a domain root or a repository subpath.

## App Routes

Routes are file-based under `src/routes`.

| File                                 | URL                    | Purpose                                |
| ------------------------------------ | ---------------------- | -------------------------------------- |
| `src/routes/index.tsx`               | `/`                    | Landing page                           |
| `src/routes/dashboard.tsx`           | `/dashboard`           | Student dashboard                      |
| `src/routes/chapters.index.tsx`      | `/chapters`            | Chapter list                           |
| `src/routes/chapters.$chapterId.tsx` | `/chapters/:chapterId` | Chapter detail, topics, and exercises  |
| `src/routes/lesson.$topicId.tsx`     | `/lesson/:topicId`     | AI lesson view                         |
| `src/routes/submit.tsx`              | `/submit`              | Answer upload and mock evaluation flow |
| `src/routes/progress.tsx`            | `/progress`            | Student progress dashboard             |
| `src/routes/admin.tsx`               | `/admin`               | Admin operations dashboard             |

`src/routeTree.gen.ts` is generated by TanStack Router. Do not edit it by hand.

## Data Model

The demo data contains:

- `Chapter`: chapter metadata, progress, topics, and exercises
- `Topic`: lesson metadata, duration, completion state, and solved examples
- `Exercise`: exercise number, question count, lock state, and optional score
- `studentProfile`: profile statistics used by dashboard and progress pages

The UI currently reads this data directly from `src/lib/mock-data.ts`. Replacing the mock data with a backend should start by preserving these shapes or introducing a typed adapter layer.

## Development Notes

- The shared app header is in `src/components/app-header.tsx`.
- Theme colors, gradients, shadows, and font tokens are defined in `src/styles.css`.
- `src/main.tsx` mounts the browser app with `RouterProvider`.
- `src/router.tsx` uses hash routing so static deep links work on GitHub Pages.
- The root route in `src/routes/__root.tsx` sets metadata, error handling, `QueryClientProvider`, and the route outlet.
- `vite.config.ts` uses explicit TanStack Router, React, Tailwind, and tsconfig-paths Vite plugins.

## Current Limitations

- Authentication is not implemented.
- Answer upload and evaluation are simulated in local component state.
- Chapter, lesson, learner, and admin data are static mock values.
- Download, camera, email, and content generation buttons are UI placeholders.
